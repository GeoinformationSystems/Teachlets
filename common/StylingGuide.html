Styling Guide for Teachlets

1. Basic structure of each teachlets index.html. Can be copied from here and used as skeleton structure for new teachlets.

	
<!DOCTYPE html>
<html>
<head>
	<meta http-equiv="Content-Type" content="text/html" charset="UTF-8">
	<link rel="stylesheet" href="../common/style.css" type="text/css">
	
	<script src="https://ajax.googleapis.com/ajax/libs/jquery/3.5.1/jquery.min.js"></script>
	<link href="https://fonts.googleapis.com/css2?family=Open+Sans&display=swap" rel="stylesheet">
	
	<title>...</title>

	<!-- include individual javascript files for specific teachlet functionality  -->
	<script src="js/app.js"></script>
	<script src="js/main.js"></script>
	
	<!-- include ESRI libraries and styling if a map is used -->
	<link rel="stylesheet" href="https://js.arcgis.com/4.17/esri/css/main.css">
	<script src="https://js.arcgis.com/4.17/"></script>
	
	<!-- include d3 library if needed for drawing -->
	<script src="https://d3js.org/d3.v3.min.js"></script>

</head>
<body>
	<div id="frame">
		<div id="header">
			<h1>...</h1>
		</div>
		<div id="info">
			<!-- up to 3 divs with following possible sizing options -->
			<div style="max-height:fit-content;">
				<!-- size of div will fit its contents -->
			</div>
			<div style="max-height:none; flex:1 1 0%;">
				<!-- div will fill out the leftover space -->
			</div>
			<div style="max-height:28vh">
				<!-- fixed maximum size, will fit its contents if smaller  -->
			</div>
		</div>
		<div id="content">
			<!-- remove canvas if map is used, map container id will be "content" -->
			<canvas id="drawfield"></canvas>
		</div>
	</div>
</body>	
</html>

2. id=header block should include <h1> with the title of the teachlet. A simple border at the bottom will be added through CSS to separate from application content. Uses full width of the viewport.

	
<h1>...</h1>

3. id=info block includes up to three <div>s with different height options depending on significance of the content.

Example 1: three blocks (two containig important content like 'Steuerung' and 'Ausgabe' | center one will fill the leftover space)

	
<div style="max-height:fit-content;">
	<!-- size of div will fit its contents -->
</div>
<div style="max-height:none; flex:1 1 0%;">
	<!-- div will fill out the leftover space -->
</div>
<div style="max-height:fit-content;">
	<!-- size of div will fit its contents -->
</div>

Example 2: two blocks (first one will fill the leftover space | second one containig important content like 'Steuerung' )

	
<div style="max-height:none; flex:1 1 0%;">
	<!-- div will fill out the leftover space -->
</div>
<div style="max-height:fit-content;">
	<!-- size of div will fit its contents -->
</div>

Example 3: two blocks (one with a fixed maximum height of 30vh | second one will fill the leftover space)

	
<div style="max-height:max-height:30vh;">
	<!-- size of div will fit its contents -->
</div>
<div style="max-height:none; flex:1 1 0%;">
	<!-- div will fill out the leftover space -->
</div>

no screenshot, currently not used in existing teachlets

4. Blocks inside of the id=info block should include a <h1> and another <div> for text content. Other used elements are: <button>, <input> or further elements needed for functionality.

	
<div style="max-height:none; flex:1 1 0%;">
	<h1>...</h1>
	<div>explanation</div>
	<button name="...">...</button>
	<input type="text" placeholder="..." id="...">
	<input type="range" min="1" max="20" step="1" value="..." id="...">
</div>

5. id=content block can include a <canvas> if drawing will be necessary. Leave this blank if you want to include a map. Resizing the canvas or map should be done on loading via javascript

	
<div id="content">
	<!-- remove canvas if map is used, map container id will be "content" -->
	<canvas id="drawfield"></canvas>
</div>
